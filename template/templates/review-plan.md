## Checklist

### 1. Code Quality
- [ ] Functions have clear single responsibility
- [ ] No code duplication — reuse existing utilities
- [ ] Consistent naming conventions followed
- [ ] Dead code removed (no commented-out blocks)

### 2. Architecture
- [ ] New code follows existing project patterns
- [ ] Proper separation of concerns (layers, modules)
- [ ] No circular dependencies introduced
- [ ] Changes are in the correct module/directory

### 3. Error Handling
- [ ] Errors are handled at appropriate boundaries
- [ ] User-facing error messages are helpful
- [ ] No swallowed errors (empty catch blocks)
- [ ] Edge cases considered (null, empty, overflow)

### 4. Security
- [ ] No secrets or credentials hardcoded
- [ ] User input validated at system boundaries
- [ ] Authentication/authorization on protected endpoints
- [ ] No SQL injection, XSS, or command injection vectors

### 5. Testing
- [ ] New features have corresponding tests
- [ ] Edge cases are tested
- [ ] Tests are deterministic (no flaky tests)

### 6. Documentation
- [ ] Public APIs have clear documentation
- [ ] Complex logic has explanatory comments
- [ ] README updated if behavior changes

## Severities

| Level | Description | Action |
|-------|------------|--------|
| **Critical** | Bugs, security, major architectural violations | Must fix before merge |
| **Warning** | Incorrect patterns, inconsistencies | Should fix |
| **Suggestion** | Minor improvements, style, optimizations | Consider fixing |
